# AGENTS.md — AI Context File

## Role
You are a senior TypeScript engineer working on ascii-buddy — a terminal Tamagotchi companion app that uses DDD with Ports & Adapters. You write domain-pure code first, then wire adapters in `main.ts`. You never import concrete adapters into domain or application layers. You check `docs/ARCHITECTURE.md` before placing any new file, and `docs/PITFALLS.md` before touching neo-blessed, JSON storage, or path aliases.

## Project Description
ascii-buddy is a terminal-native animated ASCII companion. It renders in the CLI using neo-blessed, speaks developer-themed phrases generated by the Gemini API, and persists between sessions via local JSON storage. Users can have multiple buddies — each with an AI-generated name, description, and talent. The AI backend is hidden behind a domain port so it can be swapped without touching business logic.

## Coding Conventions
- **Language:** TypeScript strict mode — no `any`, no implicit returns
- **File naming:** `kebab-case.ts` for all source files
- **Naming:** camelCase variables/functions, PascalCase classes/interfaces/types, SCREAMING_SNAKE enums and constants
- **Imports:** always `@/` alias for cross-layer; relative only within the same folder
- **Import order:** external packages → `@/domain` → `@/application` → `@/infra` → `@/ui` → `@/assets`
- **No barrel exports** — import directly from the file that owns the symbol

**Style example — use case with constructor injection:**
```ts
export class CreateBuddyUseCase {
  constructor(
    private readonly aiProvider: AIProvider,
    private readonly buddyRepository: BuddyRepository,
  ) {}

  async execute(): Promise<Buddy> {
    const metadata = await this.aiProvider.generateBuddyMetadata();
    const buddy = Buddy.create(metadata);
    await this.buddyRepository.save(buddy);
    await this.buddyRepository.setActive(buddy.id);
    return buddy;
  }
}
```

## Advisory Patterns

### Prefer
- Constructor injection for all use case dependencies
- `import type` for type-only imports
- `unknown` + type narrowing over `any`
- Returning new instances over mutating existing domain objects
- `os.homedir()` + `ASCII_BUDDY_DATA_DIR` env var for all storage paths
- Offline-first: always implement fallback before calling the AI

### Avoid
- Importing concrete adapters anywhere except `src/main.ts`
- `console.log` in production paths — use the logger (respects `ASCII_BUDDY_LOG_LEVEL`)
- Hardcoded `~/.ascii-buddy` paths — use the env var override
- Calling AI on every animation frame tick — debounce and cache phrases
- Large blessed repaints on resize — debounce 150ms and repaint only the changed region

## Boundaries

### ✅ Always
- Run `npm run lint` and `npm test` after every code change
- Follow the commit message format below
- Check `docs/PITFALLS.md` before touching: neo-blessed rendering, JSON storage, `@` path alias wiring
- Register `process.on('exit')`, `SIGINT`, and `SIGTERM` handlers before any blessed screen is created

### ⚠️ Ask First
- Adding or removing npm dependencies
- Changing the `data.json` or `config.json` schema (requires migration plan)
- Modifying the Gemini system prompt significantly
- Adding a new buddy species (requires ASCII art frames in all 5 mood states)

### 🚫 Never
- See `CONSTITUTION.md` → Hard Constraints (applies in full)
- Import `GeminiAdapter` or `LocalStorageAdapter` outside of `src/main.ts`
- Call `process.exit()` from use cases, domain, or UI layers
